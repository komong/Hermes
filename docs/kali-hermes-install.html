02 安装 Hermes Agent

推荐方式，一键安装

✅

官方一键安装脚本（推荐）—— 自动检测系统、安装依赖、配置 PATH，最省心

1 方式一：一键安装脚本（推荐） ▼ 
$ curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

安装脚本会自动：

  • 检测 Python 版本（需要 3.10+）
  • 用 pip 安装 hermes-agent 包
  • 下载 CLI 完成文件
  • 把 hermes 添加到 ~/.local/bin 并配置 PATH
2 方式二：pip 直接安装 ▼ 
$ pip install hermes-agent -U
# 或者：
$ pip3 install hermes-agent -U
3 方式三：源码安装（开发者） ▼ 
$ git clone https://github.com/NousResearch/hermes-agent.git
$ cd hermes-agent && pip install -e .
4 🔧 解决 "hermes: command not found" ▼

PATH 没配置好？手动修：

$ echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc
$ which hermes
/home/kali/.local/bin/hermes
⚠️

Kali 默认用户不是 root，所以安装路径在 ~/.local/bin 而不是 /usr/local/bin。这不需要 sudo，是正确的。

03 配置 API Key

让 Hermes 能调用 AI 模型

1 获取 API Key ▼

推荐使用 OpenRouter（一个聚合了几乎所有模型的网关）：

  • 访问 openrouter.ai/keys
  • 注册/登录账号
  • 点击 "Create Key"，复制生成的密钥（格式：sk-or-v1-xxxx）

也可以用其他 Provider：

DeepSeekplatform.deepseek.com免费额度充足
Groqconsole.groq.com免费，速度快
2 配置 Key（交互式向导） ▼ 
$ hermes setup
Welcome to Hermes Agent setup!

What would you like to configure?
  [1] Model & Provider
  [2] Terminal settings
  [3] Gateway settings
  [4] Tools
  [5] Agent settings
  [6] All of the above

Select option:
# 输入 1，然后输入你的 OpenRouter API Key
💡

自动路由： hermes config set 会自动把密钥放到 ~/.hermes/.env，把开关放到 ~/.hermes/config.yaml，不需要手动编辑。

3 手动配置（备选） ▼ 
$ echo 'OPENROUTER_API_KEY=sk-or-v1-your-key-here' >> ~/.hermes/.env
$ hermes config set model.provider openrouter
$ cat ~/.hermes/.env
OPENROUTER_API_KEY=sk-or-v1-your-key-here

09 🌐 配置 Web UI（Open WebUI）

通过浏览器界面与 Hermes Agent 交互，界面类似你现在看到的这个页面

💡

原理说明： Hermes Agent 内置了一个 OpenAI 兼容的 API Server（端口 8642）。任何兼容 OpenAI 格式的前端都可以连接它。Open WebUI（GitHub 126k ⭐）是最流行的自托管 AI 聊天界面，Docker 一键部署，把 Hermes Agent 当作后端大脑来用。

1 🏗️ 架构说明 ▼

数据流向：

┌──────────────────────────────────────────────────────────┐
│                                                          │
│   你（浏览器）──port 3000──▶  Open WebUI（Docker）        │
│                                       │                  │
│                                       │ http://host.docker│
│                                       │ .internal:8642/v1 │
│                                       ▼                  │
│                              ┌─────────────────────┐    │
│                              │  Hermes Agent       │    │
│                              │  Gateway + API Server│    │
│                              │      :8642           │    │
│                              └─────────────────────┘    │
│                                       │                  │
│                              调用工具：terminal / file /  │
│                              web / memory / skills / MCP  │
└──────────────────────────────────────────────────────────┘

Open WebUI 负责渲染漂亮的聊天界面，Hermes Agent 负责大脑逻辑（执行命令、读写文件、搜索等），两者通过 /v1/chat/completions API 通信。

2 🚀 启用 Hermes API Server ▼

API Server 是 Hermes 的 HTTP 接口，Open WebUI 通过它调用 Agent。两行命令搞定：

$ hermes config set API_SERVER_ENABLED true
$ hermes config set API_SERVER_KEY my-kali-secret

# 如果 Gateway 已在运行，重启使配置生效：
$ hermes gateway stop && hermes gateway
💡

hermes config set 自动把开关写到 config.yaml，把密钥写到 ~/.hermes/.env。

启动后应该看到：

[API Server] API server listening on http://127.0.0.1:8642
3 🔍 验证 API Server 正常 ▼ 
# 健康检查
$ curl -s http://127.0.0.1:8642/health
{"status":"ok",...}

# 查看可用模型
$ curl -s -H "Authorization: Bearer my-kali-secret" \
  http://127.0.0.1:8642/v1/models
{"object":"list","data":[{"id":"hermes-agent",...}]}

两条都有正常返回 → API Server 就绪！

4 🐳 安装 Docker（如未安装） ▼

Open WebUI 通过 Docker 运行，Kali 上先装 Docker：

$ sudo apt update && sudo apt install -y docker.io
$ sudo systemctl enable docker
$ sudo systemctl start docker
# 把当前用户加入 docker 组（免 sudo 运行 docker）：
$ sudo usermod -aG docker $USER && newgrp docker
# 验证：
$ docker --version
Docker version 24.x.x
⚠️

加入 docker 组后需要重新登录终端（或运行 newgrp docker）才能生效。Kali 2025 默认用户已经是 sudo 成员，所以这里不需要 root。

5 🐳 启动 Open WebUI（Docker 一条命令） ▼

在 Kali 终端运行这条命令：

$ docker run -d \
  --name open-webui \
  --restart always \
  -p 3000:8080 \
  -e OPENAI_API_BASE_URL=http://host.docker.internal:8642/v1 \
  -e OPENAI_API_KEY=my-kali-secret \
  -e ENABLE_OLLAMA_API=false \
  --add-host=host.docker.internal:host-gateway \
  -v open-webui-data:/app/backend/data \
  ghcr.io/open-webui/open-webui:main
💡

参数说明：

  • --add-host=host.docker.internal:host-gateway — Docker 容器内部访问宿主机端口的关键！
  • ENABLE_OLLAMA_API=false — 禁用默认 Ollama 后端，避免模型选择器出现空列表
  • -v open-webui-data:/app/backend/data — 持久化存储（用户账号、聊天记录）
  • --restart always — 开机自启

首次启动需要 15-30 秒下载 embedding 模型，查看进度：

$ docker logs -f open-webui
# 看到以下内容后表示启动完成：
Uvicorn running on http://0.0.0.0:8080
6 📄 Docker Compose 方式（推荐永久部署） ▼

创建 ~/hermes-web/docker-compose.yml：

services:
  open-webui:
    image: ghcr.io/open-webui/open-webui:main
    container_name: open-webui
    restart: always
    ports:
      - "3000:8080"
    environment:
      - OPENAI_API_BASE_URL=http://host.docker.internal:8642/v1
      - OPENAI_API_KEY=my-kali-secret
      - ENABLE_OLLAMA_API=false
    extra_hosts:
      - "host.docker.internal:host-gateway"
    volumes:
      - open-webui-data:/app/backend/data

volumes:
  open-webui-data:
$ mkdir -p ~/hermes-web && cd ~/hermes-web
$ nano docker-compose.yml  # 粘贴上面的内容
$ docker compose up -d
# 以后重启也只需要：
$ docker compose up -d

8 🛡️ Kali 特殊注意事项 ▼
问题解决方案
Docker daemon 未启动sudo systemctl start docker
普通用户无法运行 dockersudo usermod -aG docker $USER && newgrp docker
docker-compose 太旧sudo apt install docker-compose-v2 或用内置的 docker compose（v2）
端口 3000 被占用改端口：-p 8080:8080，访问 http://localhost:8080
内存不足（Open WebUI 需要 ~4GB）给 VM 分配更多内存，或 --memory=4g 限制 Docker
host.docker.internal 不解析确认命令里加了 --add-host=host.docker.internal:host-gateway
想从宿主机访问 Docker 内的 Hermes先 hermes gateway 在宿主机上跑起来，容器通过 host.docker.internal 访问
9 👥 多用户配置（Profiles） ▼

给不同用户分配独立的 Agent 实例（独立配置、记忆、技能）：

# 创建两个独立实例
$ hermes profile create alice
$ hermes -p alice config set API_SERVER_ENABLED true
$ hermes -p alice config set API_SERVER_PORT 8643
$ hermes -p alice config set API_SERVER_KEY alice-secret

$ hermes profile create bob
$ hermes -p bob config set API_SERVER_ENABLED true
$ hermes -p bob config set API_SERVER_PORT 8644
$ hermes -p bob config set API_SERVER_KEY bob-secret

# 启动两个 Gateway
$ hermes -p alice gateway &
$ hermes -p bob gateway &

然后在 Open WebUI 的 Admin Settings → Connections 中分别添加两个连接，模型下拉菜单里就能看到 alice 和 bob 两个独立 Agent。

10 ❓ 常见问题

FAQ

Q: 安装需要 root/sudo 吗？▼

不需要。 Hermes 安装在用户目录 (~/.local/bin)，配置文件在 ~/.hermes/。Docker 部分需要 sudo 安装 docker.io，但日常运行不需要。

Q: Hermes 可以调用 Kali 工具（Nmap、Metasploit 等）吗？▼

可以！通过 terminal 工具集直接调用任何命令行工具：nmap -sV target.com。也可以通过 MCP 接口连接 Nmap、Metasploit 等已有 MCP 服务器。

Q: Kali 在虚拟机中运行，Hermes 可以吗？▼

可以。VMware、VirtualBox、WSL2 中的 Kali 都可以。唯一要求是网络连通性（能访问模型提供商的 API）。如果网络受限，可以配置本地模型通过 llama.cpp 运行。

Q: Hermes 能完全替代命令行操作吗？▼

大部分场景可以。它可以执行任意 Shell 命令、读写文件、搜索网页、写代码并运行测试。但它不是渗透测试工具的替代品——它是帮你更高效使用这些工具的助手。

Q: 如何排查问题？▼
$ hermes doctor --fix
$ tail -50 ~/.hermes/logs/gateway.log
$ grep -i error ~/.hermes/logs/gateway.log | tail -20
Q: 能用本地模型（Ollama）吗？▼

可以！通过自定义 Provider 端点配置 Ollama：

$ hermes config set model.provider custom
$ hermes config set model.base_url http://localhost:11434/v1
$ hermes config set model.api_key ollama
$ hermes model llama3.2
Q: Web UI 和命令行有什么区别？▼

功能完全一样！Web UI（Open WebUI）只是换了一个漂亮的浏览器界面，背后调用的是同一个 Hermes Agent，工具集、记忆、技能都是共享的。适合不想用终端、喜欢图形界面的人。

Q: Open WebUI 首次启动很慢怎么办？▼

首次启动需要下载 embedding 模型（约 150MB）。如果网络慢，可以：
1. 配置代理：export HTTP_PROXY=... 再运行 docker
2. 手动 pull：docker pull ghcr.io/open-webui/open-webui:main
查看下载进度：docker logs -f open-webui
